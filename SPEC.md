# クイズ得点アプリ — デザイン全面刷新 指示書（Gemini Canvas 用）

## 0. これは何 / Geminiへの最重要指示
クイズ大会用のリアルタイム得点表アプリ（既に稼働中）の **UI（見た目）だけをゼロから大胆に作り直す**
ための指示書です。

- ✅ **現在の見た目は一切踏襲しないでよい。大胆に作り直してください。**「ちょっと色を変える」ではなく、
  レイアウト・コンポーネント・装飾を新しく設計してOK。
- ✅ **モックデータ（仮のプレイヤー/チーム/得点）でUIだけ作って構いません。** バックエンド接続は不要。
  後で開発者が既存のロジック（Firebase通信・得点計算）に繋ぎ直します。
- ✅ 出力は **React 単一ファイル（Tailwind CSS, ビルド不要のCDN前提）** が望ましい。
- ⚠️ ただし後で繋ぎ直せるよう、**「6. コンポーネント分割」「7. データの形」の“名前と構造”は維持**してください
  （見た目は自由、props名・state構造・コンポーネントの役割分担は保つ）。

---

## 1. デザイン方針：明るくポップ
- **ライト基調**（白〜淡色背景）で親しみやすく、丸み（rounded-2xl等）・やわらかい配色。
- ポップで楽しいが、**得点と順位がはっきり読める**こと（数字大きめ・コントラスト確保）。
- **モバイルファースト**（スマホ縦持ち中心）。ボタンは大きく押しやすく。
- クイズらしい軽い高揚感（順位変動のアニメ、1位の演出など）はOK。ただし過剰な点滅は避ける。
- 配色・フォント・装飾・アイコンは自由に提案してよい。

---

## 2. 登場人物
- **ホスト**: 1名。部屋を作り、得点を操作する司会者。
- **参加者**: 複数。招待リンク/部屋コードで入室し、自分のスマホで全員の得点を見る（設定により自分の得点操作も可）。

---

## 3. 画面一覧（作ってほしい画面）
1. **ホーム(home)**: 「部屋を作る（ホスト）」ボタン／「部屋コード(数字4桁)＋名前で参加」フォーム。
2. **入室(join)**: 招待リンクで来たとき。部屋コードを表示し「名前を入力して入室」。
3. **ホスト画面(host)**: 上部に **CODE（数字4桁）** と「リンクコピー」「退出」。**「ランキング/グラフ」タブ**。
   - ランキング: チーム得点ランキング＋プレイヤー一覧（各行に得点操作ボタン）。
   - グラフ: 得点推移の折れ線グラフ。
   - チーム管理パネル（PCはサイドバー、スマホは「👥チーム」ボタンで開くシート）。
   - 下部に操作バー（得点モード切替・1つ戻す・全員1に・0リセット）。
4. **参加者画面(participant)**: 上部に CODE と「退出」。「ランキング/グラフ」タブ。閲覧専用の得点表
   （順位・1位に王冠・自分の行を強調・チーム名チップ）。設定ONなら下部に「自分の得点だけ操作するバー」。
5. **状態表示**: 「接続中…」バナー、「Firebase未設定」バナー、「部屋が見つからない」エラー。

---

## 4. 機能インベントリ（v21・全機能）
UIとして次をすべて表現してください（ロジックは既存のものを使うので、見た目と操作要素があればOK）。

- 部屋作成、共有リンク/部屋コード（**数字4桁**）、コピー、退出（タイトルへ戻る）。
- **得点操作モード**（ホスト＆許可された参加者）:
  - ○×（○=＋規定点／×=−規定点）
  - 加点のみ（○=＋規定点）
  - 自由計算（＋ − × ÷、値は `2` / `0.5` / `1/2` など小数・分数可）
  - クイック（＋1〜＋5 ／ −1 のボタン）
- **得点表示モード**（部屋全体の切替）:
  - **＋M/−N**: 合計1つの数値スコア。
  - **Simple**: ○の数と×の数を別々にカウントし「○ － ×」（例 3－1）で表示。並び順は○の多い順。
- **チーム機能**: 作成・改名・削除、メンバーを**ドラッグ＆ドロップ**で割当（未所属⇄チーム）、
  チーム得点を **積／和** で切替表示。Simpleモードのチームは「○合計 － ×合計」。
- **参加者採点**: ホストがONにすると、参加者は**自分の得点だけ**を全モードで操作可。
- **1つ戻す(Undo)**、**全員1に**、**全員0リセット**、プレイヤーの改名/削除。
- **得点推移グラフ**（プレイヤーごとの折れ線＋凡例）。
- 1位ハイライト（王冠）、自分の行ハイライト、チーム名チップ、順位表示。

---

## 5. 状態（state）の形 ※この構造でモックを作ってください
```js
state = {
  roomId: "0427",
  scoreMode: "addsub",        // "addsub" | "simple"
  teamMode: "product",        // "product" | "sum"
  guestScoring: false,        // 参加者採点ON/OFF
  players: [
    { id, name, score: "10/1", maru: 0, batsu: 0, teamId: "t1"|null }
  ],
  teams: [ { id, name } ],
  timeline: [ { t: 1234, scores: { [playerId]: 5 } } ]  // グラフ用
}
```
- 得点 `score` は **"分子/分母" の文字列**（例 "10/1"=10, "1/2"=0.5）。表示は整形関数で数値化される前提でOK
  （モックでは見やすい数字を入れてよい）。
- Simpleモードは各プレイヤーの `maru`(○の数) / `batsu`(×の数) を使う。

---

## 6. コンポーネント分割（役割は維持・見た目は自由）
次の単位で組んでください（名前は合わせてくれると統合が楽です）:
- `HomeView` / `JoinView` / `HostView` / `ParticipantView`
- `TeamBoard`（チーム得点ランキング）
- `PlayerList` ＋ プレイヤー行（順位・名前・得点・操作ボタン）
- `ScoreGraph`（折れ線グラフ）
- 得点操作ボタン群（○×・加点・計算・クイック・Simpleの○/×）

操作は「コールバックを呼ぶだけ」にしておいてください（例: `onScore(playerId, '+', '10')`、
`onMaru(playerId)`、`onAssignTeam(playerId, teamId)` 等）。中身は開発者が既存ロジックに接続します。

- やらなくてよい: Firebase接続・通信・得点計算・データ保存（既存実装を使うため）。
