# quiz_point — クイズ用得点サイト（tokuten 拡張版）

クイズ大会用のリアルタイム得点表。既存の「tokuten」が「自分の得点 ± x 点」しかできないのに対し、
**かけ算・わり算（分数・小数倍を含む）** と **チーム得点（メンバーの積）** に対応しています。

ホストが部屋を建て、共有リンクを配布。参加者は各自のスマホから名前を入れて入室し、
**全員の得点をリアルタイムに閲覧**できます（操作はホストのみ）。

## 主な機能

- ホストが部屋を作成 → **共有リンク / 部屋コード**を発行
- 参加者はリンクから **名前を設定して入室 → 閲覧専用**（自分の行をハイライト）
- ホスト操作: 各プレイヤーの得点に **＋ − × ÷**（値は `2` / `0.5` / `1/2` のように小数・分数可）
- **全員を 1 にする** ボタン / **全員 0 にリセット**（初期得点はデフォルト 0）
- **↩ 一つ戻す（Undo）**：直前の操作を取り消し
- 得点表は **大きい順（降順）** に自動ソート、1位をハイライト
- **チーム機能**：チーム作成・メンバー割当・**チーム得点 =（メンバー各得点の積）** を表示
  - 例: メンバーが 2,1,2,1,2 点 → チーム得点は `2×1×2×1×2 = 8`
- **得点は内部で有理数（分数）として厳密に保持**し、表示時のみ小数へ丸める
  - 例: `÷3` してから `×3` しても誤差なく元の値に戻る
- 全デバイスへ Socket.IO で即時同期

## ローカルで動かす

```bash
npm install
npm start
# http://localhost:3000 を開く
```

ポートは環境変数 `PORT` で変更できます（既定 3000）。

## テスト

得点計算ロジック（`src/scoreEngine.js`）のユニットテスト:

```bash
npm test
```

## 使い方

1. トップページで **「部屋を作る（ホスト）」** を押す → ホスト操作パネルへ。
2. パネル上部の **共有リンク**（または部屋コード）を LINE 等で参加者に共有。
3. 参加者はリンクを開き、**名前を入力して入室**。以降は得点表を閲覧するだけ。
4. ホストは各プレイヤー行で演算子（＋−×÷）と値を選んで **適用**。必要に応じて
   「全員を1に」「リセット」「一つ戻す」やチーム編成を行う。

> ホスト権限はブラウザの localStorage に保存されます。部屋を作成したブラウザから
> `host.html` を開くと操作を続行できます（別ブラウザ／別端末からは操作できません）。

## デプロイ

サーバ1つで完結します。Node が動くホスティングならどこでも可。

### Render（例）

- リポジトリを連携（`render.yaml` 同梱）
- Build: `npm install` / Start: `npm start`（`PORT` は自動注入）
- 発行された URL を共有すれば、参加者はリンクから入室できます。

その他 Heroku 系は `Procfile`（`web: npm start`）に対応。

> ⚠️ 部屋の状態はサーバのメモリ上に保持します。無料プランのスリープやサーバ再起動が
> 起こると部屋は消えます（イベント中の連続稼働を想定）。

## 構成

```
server.js                サーバ（静的配信 + Socket.IO + 部屋ロジック）
src/scoreEngine.js       得点計算の純粋ロジック（パース／演算／チーム積／整形）
public/index.html        トップ / 参加者の入室・閲覧画面
public/host.html         ホスト操作パネル
public/js/common.js      共有ユーティリティ
public/js/participant.js 参加者ロジック
public/js/host.js        ホストロジック
public/css/style.css     スタイル
test/scoreEngine.test.js テスト
```
